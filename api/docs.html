hockey-LIVE.sk API Documentation

Overview

The hockey-LIVE.sk API provides access to various hockey-related information including team statistics, player statistics, game reports, game lists, and team standings for top hockey leagues and tournaments like NHL, IIHF World Championships, Olympic Games, KHL and Slovak extraliga. The API is designed to deliver data in a simple and efficient manner, allowing developers to integrate hockey-related information into their applications.

API key

For obtaining a free API key for our API, please contact us by e-mail.

Base URL

https://hockey-live-sk-data.p.rapidapi.com

Team Statistics

Get Team Statistics by Team, League, and Year

GET /team/{team}/{league}/{year}?key={API_key}
  • team: 3-char country ISO code (e.g., SVK).
  • league: League identifier (WCH for World Championship, OG for Olympic Games, NHL for NHL season, KHL for KHL season, EL for Slovak extraliga season).
  • year: Tournament year or season (e.g., 2018 for the year 2018; or 2018/2019 for NHL, KHL and EL season).
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/team/SVK/WCH/2018?key=your_api_key

Player Statistics

Get Player Statistics by ID

GET /player/{id}?key={API_key}
  • id: Player ID.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/player/2620?key=your_api_key

Get Player Statistics by Name and League

GET /player/{player_name}/{league}?key={API_key}
  • player_name: Player's name.
  • league: League identifier.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/player/SATAN%20Miroslav/NHL?key=your_api_key

Game Information

Get Game Report by ID

GET /game/{id}?key={API_key}
  • id: Game ID.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/game/20380?key=your_api_key

Get Last Game Report by Teams and League

GET /game/{team1}/{team2}/{league}?key={API_key}
  • team1: 3-char country ISO code for the home team.
  • team2: 3-char country ISO code for the away team.
  • league: League identifier.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/game/SVK/CAN/WCH?key=your_api_key

Game List

Get Game List by League and Year

GET /games/{league}/{year}?key={API_key}
  • league: League identifier.
  • year: Tournament year or season.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/games/NHL/2020?key=your_api_key

Team Standings

Get Team Standings by League and Year

GET /table/{league}/{year}?key={API_key}
  • league: League identifier.
  • year: Tournament year or season.
  • API_key: Your API key for authentication.

Example:

https://hockey-live-sk-data.p.rapidapi.com/table/KHL/2021?key=your_api_key

Authentication

All requests must include the key parameter with your unique API key for authentication.

Response Format

The API returns data in JSON format.

Response Parameters

Most of the return parameters should be self-explanatory. Some that could be unclear are listed here:

  • po_type = stage of playoff the game is played in (possible values are: NULL-no playoff, Q-qualification round, QF-quarterfinals, SF-semifinals, B-bronze medal game, F-finals, SC-Stanley Cup/Gagarin Cup finals)
  • status = status in which the game is currently in (possible values are: final result, scheduled, get ready, in the 1st period, after 1st period, in the 2nd period, after 2nd period, in the 3rd period, after 3rd period, overtime, after overtime)
  • when = when was the goal scored (possible values are: normal, PP1-5on4 powerplay, PP2-5on3 powerplay, SH1-4on5 shorthanded, SH2-3on5 shorthanded, PS-penalty shot)
  • where = where is the game played (possible values are: home, away)
  • result = outcome of the game for the team (possible values are: loss, win)
  • overtime = whether the game ended in overtime (possible values are: 0-ended in regulation time, 1-ended in overtime, 2-ended in shootouts)
  • group = group for the team in team standings (possible values are: eg. A, B, ...)
  • pos = position the players plays in (possible values are: GK-goalkeeper, F-forward, D-defense, RW-right wing, LW-left wing, CE-center, RD-right defense, LD-left defense)
  • clinch = whether the team clinched the quarterfinals (possible values are: t칤m u m치 zaisten칰 칰캜as콘 vo 코tvr콘fin치le-clinched quarterfinals, t칤m sa u nedostane do 코tvr콘fin치le-cannot make quarterfinals, t칤m zostupuje do I.DIV-team is dropping to the first division)
